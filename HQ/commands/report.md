# /report <clientId>

## Validation

1. **clientId is required** — abort if not provided
2. **Verify client exists** in `HQ/crm/clients.json`
3. **Verify client status is "operate"** — no reports for clients still in build phase

## Steps

### 1. Get Client Details
Read `HQ/crm/clients/<clientId>/profile.md` for:
- Vercel project name
- App name
- Domain

### 2. Scrape Vercel Analytics
Use Playwright to:
1. Navigate to the Vercel Analytics dashboard for the client's project
2. Capture: visitors, page views, top pages, performance scores (LCP, FID, CLS)
3. Screenshot the dashboard for reference

### 3. Generate Report
Format into a branded PDF structure:

```markdown
# ZeroEn Monthly Analytics Report
## [App Name] — [Month YYYY]

### Traffic Overview
| Metric | This Month | Last Month | Change |
|--------|-----------|-----------|--------|
| Visitors | X | X | +/-% |
| Page Views | X | X | +/-% |
| Avg. Session | Xs | Xs | +/-% |

### Top Pages
| Page | Views | % of Total |
|------|-------|-----------|
| / | X | X% |

### Performance (Core Web Vitals)
| Metric | Score | Rating |
|--------|-------|--------|
| LCP | Xs | Good/Needs Work/Poor |
| FID | Xms | Good/Needs Work/Poor |
| CLS | X | Good/Needs Work/Poor |

### Recommendations
[1-3 actionable suggestions based on the data]

---
Generated by ZeroEn | [date]
```

### 4. Update CRM
Update `HQ/crm/clients/<clientId>/profile.md`:
- Set `Last report sent` to today
- Set `Next report due` to next month

### 5. Deliver
- Save report to `HQ/crm/clients/<clientId>/reports/[YYYY-MM].md`
- Notify operator for review before sending to client
